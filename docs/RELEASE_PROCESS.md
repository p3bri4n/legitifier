# Release process

## Pre-release checklist

- [ ] All P0 issues closed
- [ ] `make test` and `make lint` pass on `main`
- [ ] `pip-audit --requirement requirements.lock --strict` passes
- [ ] CHANGELOG.md updated
- [ ] Version bumped via `make bump`
- [ ] No PR merged in last 24h (cooldown for supply-chain safety)

## Release steps

1. `git checkout -b release/<version>`
2. `make bump`
3. Update CHANGELOG.md with the version entry
4. Open PR, get 2 approvals (once multi-maintainer)
5. Merge to `main`
6. `git tag <version> && git push --tags`
7. CI publishes via Trusted Publishing (OIDC — no API key needed)
8. Verify attestation: `gh attestation verify dist/legitifier-<version>-*.whl --owner p3bri4n`
9. Create GitHub release with SBOM (sbom.cyclonedx.json) attached

## Rollback

1. `pip install pypi-yank && pypi-yank legitifier <version>`
2. Open a GitHub Security Advisory describing the issue
3. Publish a patched release following the steps above
4. Notify via the channel described in SECURITY.md

## Notes

- PyPI Trusted Publishing is configured at https://pypi.org/manage/account/publishing/
- The OIDC token is scoped to the `release.yml` workflow only
- SBOM is generated by syft (CycloneDX JSON) and attached to the GitHub release
- Build provenance is attested via `actions/attest-build-provenance`
